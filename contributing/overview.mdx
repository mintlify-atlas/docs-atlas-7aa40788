Welcome to the **e** project - a delightful collection of programs that print the letter 'e' in 130+ programming languages and file formats!

We're thrilled that you're interested in contributing. This project celebrates the diversity of programming languages and the creativity of developers worldwide. Whether you're a seasoned polyglot programmer or just starting out, there's a place for you here.

## Why Contribute?

- **Learn new languages**: Explore how different programming languages work
- **Join a fun community**: Participate in a lighthearted, creative project
- **Practice your skills**: Write concise, creative code
- **Build your portfolio**: Show off your knowledge of multiple languages

## Types of Contributions

We welcome several types of contributions:

### Adding New Language Implementations

The most common contribution! Add a new program that prints 'e' in a language we don't have yet. See our [Adding Languages Guide](/contributing/adding-languages) for detailed instructions.

**Examples of great contributions:**
- Adding a new programming language (e.g., `e.zig`, `e.kt`)
- Adding a new file format (e.g., `e.svg`, `e.json`)
- Adding platform-specific variants (e.g., `e_macOS_64.S`, `e_win32.asm`)

### Improving Existing Implementations

Make existing implementations more interesting, efficient, or idiomatic:
- Simplify code while maintaining functionality
- Add creative variations (like `e.jsfuck.js`)
- Fix bugs or compatibility issues
- Improve code style to match language conventions

### Documentation Improvements

Help others understand the project:
- Fix typos or unclear explanations
- Add examples or clarifications
- Translate documentation
- Improve setup instructions

### Infrastructure & Tooling

Enhance the project infrastructure:
- Improve build scripts
- Add testing utilities
- Create automation tools
- Enhance CI/CD pipelines

## What Makes a Good Contribution?

The best contributions to the **e** project:

1. **Print 'e' infinitely**: Programs should continuously output the letter 'e'
2. **Follow language conventions**: Use idiomatic style for each language
3. **Are creative**: Don't be afraid to have fun! (see `e.jsfuck.js` or `e.emojic`)
4. **Use standard file extensions**: Name files with the standard extension for the language
5. **Work correctly**: Test your implementation before submitting
