The **e** project demonstrates the remarkable diversity of digital file formats by encoding the simple letter 'e' in over 130 different ways. From programming languages to images, audio files to compressed archives, each format has its own unique approach to representing this single character.

## Format Categories

The project spans multiple categories of file formats:

### Programming Languages
Over 100 programming languages, including C, Python, JavaScript, Rust, Haskell, and many esoteric languages like Brainfuck and ArnoldC. Each prints the letter 'e' to standard output.

### Image Formats
Visual representations of 'e' in various image formats:
- **Raster formats**: PNG, JPEG, BMP
- **Vector formats**: SVG
- File sizes range from 409 bytes (SVG) to 418 KB (JPEG)

See the [Images](/formats/images) page for details.

### Audio Formats
Audio recordings of the letter 'e' being spoken:
- **WAV**: Uncompressed audio (642 KB)
- **MP3**: Compressed audio (26 KB)

See the [Audio](/formats/audio) page for details.

### Compressed Archives
Demonstrating compression algorithms:
- **Gzip**: 4.7 MB (multiple 'e's compressed)
- **XZ**: 153 KB

See the [Archives](/formats/archives) page for details.

### Document Formats
Typeset and formatted documents containing 'e':
- **LaTeX**: Source code for typesetting
- **PDF**: Rendered portable document
- **ODT**: OpenDocument text format
- **PostScript**: Page description language

See the [Documents](/formats/documents) page for details.

### Data Formats
Structured data representations:
- **JSON**: `{"e": "e"}`
- **YAML**: `e: e`
- **TOML**: `[E]` with `e = "e"`
- **HTML**: Interactive page that continuously appends 'e'
- **XML**: Structured markup
- **Markdown**: Text with 'e' characters

## Design Philosophy

Each implementation follows a simple principle: **output or display the letter 'e'**. However, the way this is achieved varies dramatically:

- **Text-based formats** store 'e' as character data
- **Binary formats** encode 'e' in their specific binary structure
- **Image formats** render 'e' as pixels
- **Audio formats** record the sound of 'e' being pronounced
- **Compression formats** demonstrate how 'e' (or multiple 'e's) can be compressed
- **Programming languages** execute code to print 'e'

## File Size Comparison

The representation of 'e' varies dramatically in size depending on the format:

| Format | Size | Notes |
|--------|------|-------|
| Plain text | 1-2 bytes | Just the character |
| SVG | 409 bytes | Vector graphics markup |
| JSON | 15 bytes | `{"e": "e"}` with formatting |
| YAML | 3 bytes | `e: e` |
| PNG | 3.8 KB | Raster image |
| PDF | 8.6 KB | Formatted document |
| ODT | 9.7 KB | OpenDocument (ZIP-based) |
| MP3 | 26 KB | Compressed audio |
| BMP | 45 KB | Uncompressed bitmap |
| XZ archive | 153 KB | Compressed data |
| JPEG | 418 KB | High-resolution image |
| WAV | 642 KB | Uncompressed audio |
| Gzip | 4.7 MB | Many 'e's compressed |

## Viewing the Files

All source files are available in the project repository. Each format can be viewed using appropriate tools:

- **Images**: Any image viewer or web browser
- **Audio**: Media players (VLC, mpv, browser audio element)
- **Documents**: PDF readers, LibreOffice, text editors
- **Archives**: `gzip`, `xz`, `tar` command-line tools
- **Data formats**: Text editors, browsers (for HTML)
- **Programming languages**: Respective language interpreters/compilers

## Contributing

The project welcomes contributions of 'e' implementations in additional file formats and programming languages. Each contribution should follow the simple principle: represent or output the letter 'e' in the most idiomatic way for that format.
