Compiled languages transform source code into machine code before execution, offering performance and type safety. Here's how different compiled languages print the letter 'e'.

#define eeeeeeeeeee return
#define eeeeeeeeeeee on_exit

## Rust

Rust offers two implementations - functional and imperative:

<CodeGroup>
```rust Functional (with feature flag)
#![feature(main)]
#[cfg(feature = "functional")]
#[main]
fn e() {
    std::iter::repeat('e').for_each(|e| print!("{}", e));
}
```

```rust Imperative (default)
#![feature(main)]
#[cfg(not(feature = "functional"))]
#[main]
fn e() -> ! {
    loop {
        print!("e");
    }
}
```
</CodeGroup>

The Rust implementation showcases:
- Feature flags for conditional compilation
- Type-safe infinite loops with `-> !` (never type)
- Iterator chains in the functional style

## Go

Go's implementation is remarkably concise:

```go e.go
package main

func main() {
	for {
		print("e")
	}
}
```

<Note>
Go's `print` is a built-in function, not requiring any imports for basic output.
</Note>

## System Languages

<Tabs>
  <Tab title="D">
    ```d e.d
    import std.stdio;

    int main() {
       for(;;) {
          writefln("e");
       }
       return 0;
    }
    ```
  </Tab>
  <Tab title="Nim">
    ```nim e.nim
    while true: echo 'e'
    ```
    
    Nim's clean syntax makes it one of the most readable implementations.
  </Tab>
  <Tab title="Zig">
    ```zig e.zig
    const std = @import("std");

    pub fn main() !void {
        var stdout_file = try std.io.getStdOut();
        while (true) {
            try stdout_file.write("e");
        }
    }
    ```
    
    Zig requires explicit error handling even for standard output.
  </Tab>
</Tabs>
