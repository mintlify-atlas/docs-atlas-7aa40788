Functional programming languages emphasize immutability, pure functions, and recursion. Here's how they approach the infinite task of printing 'e'.

## Pure Functional

### Haskell

Haskell's elegant solution using infinite lists:

```haskell e.hs
main = putStr $ cycle "e"
```

This one-liner demonstrates:
- `cycle` creates an infinite list by repeating its argument
- `putStr` outputs the string without a newline
- Lazy evaluation allows working with infinite data structures

<Note>
Haskell's lazy evaluation means the infinite list is generated on-demand, not stored in memory.
</Note>

### OCaml

OCaml with tail-recursive function:

```ocaml e.ml
let rec eee e = Printf.printf "e"; eee ();;
let () = eee ();;
```

The implementation:
- Uses tail recursion for constant stack space
- `rec` keyword allows the function to call itself
- Double semicolon `;;` terminates top-level statements

## Functional Features in Multi-Paradigm Languages

### Scala

Scala's functional approach using streams:

```scala e.scala
object e extends App {
  Stream.continually("e").foreach(print)
}
```

<Accordion title="Scala Implementation Details">
- `Stream.continually` creates an infinite lazy stream
- `foreach` applies the function to each element
- `extends App` provides a main entry point without boilerplate
- Lazy evaluation prevents memory exhaustion
</Accordion>

### Rust (Functional Style)

Rust's iterator chains:

```rust e.rs
std::iter::repeat('e').for_each(|e| print!("{}", e));
```

Features:
- `repeat` creates an infinite iterator
- `for_each` is a consuming iterator method
- Zero-cost abstractions - compiles to efficient code

## Concatenative Languages

### Forth

Stack-based programming:

```forth e.fs
: e begin ." e" again ;
e
```

- `: e` defines a new word named 'e'
- `begin ... again` creates an infinite loop
- `."` prints a string literal

## Logic Programming

### Prolog

```prolog e.pro
e :- write('e'), e.
:- e.
```

Prolog's approach:
- Defines a rule that writes 'e' and calls itself
- Uses backtracking and unification
- The `:-` operator separates head from body

## Functional Concepts Demonstrated

<CardGroup cols={2}>
  <Card title="Tail Recursion" icon="rotate">

## Performance Considerations

<AccordionGroup>
  <Accordion title="Tail Call Optimization">
    Languages with guaranteed TCO (Scheme, Erlang) maintain constant stack space for recursive implementations.
    
    Without TCO, stack overflow eventually occurs.
  </Accordion>
  <Accordion title="Lazy vs Eager Evaluation">
    Lazy languages (Haskell) can work with infinite structures efficiently.
    
    Eager languages (OCaml) must use explicit recursion.
  </Accordion>
  <Accordion title="Compilation vs Interpretation">
    Compiled functional languages (OCaml, Haskell with GHC) approach C-like performance.
    
    Interpreted Lisps have more overhead but still perform well.
  </Accordion>
</AccordionGroup>

<Tip>
Functional approaches often lead to more concise code. Haskell's one-liner is among the shortest implementations across all languages.
</Tip>
