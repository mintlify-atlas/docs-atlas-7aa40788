# The Philosophy of 'e'

## What is 'e'?

At its surface, 'e' is simple: a collection of programs across 130+ languages that all do one thing—print the letter 'e' infinitely.

But beneath this simplicity lies a profound statement about the nature of programming, computation, and the diversity of human expression through code.

<Note>
  'e' is not just about the output. It's about the journey—the myriad ways different languages, paradigms, and programmers express the same fundamental idea.
</Note>

## Minimalism as Art

The 'e' project embraces minimalism not as a constraint, but as a canvas. By fixing the output, we highlight the differences in how languages think about:

- **Control flow**: Loops, recursion, iteration
- **I/O**: stdout, print functions, system calls
- **Type systems**: Strings, characters, symbols
- **Paradigms**: Imperative, functional, declarative, esoteric

## A Living Archive

The 'e' repository serves as:

### 1. A Language Museum

From COBOL to Rust, from Assembly to WebAssembly, 'e' documents the evolution of programming languages across decades. Each implementation is a time capsule of how that language thinks about the most basic computational task.

### 2. A Learning Resource

Want to learn a new language? Start by reading its 'e' implementation. You'll immediately understand:
- How to write a loop
- How to print to stdout
- The basic syntax and structure
- The language's philosophy

### 3. A Rosetta Stone

Comparing implementations side-by-side reveals deep insights about language design:

<CodeGroup>

```go Go - Simplicity
package main

func main() {
	for {
		print("e")
	}
}
```

```rust Rust - Safety
#![feature(main)]
#[main]
fn e() -> ! {
    loop {
        print!("e");
    }
}
```

```java Java - Verbosity
public class e {

	public static void main(String[] e) {
		while("e".equals("e")) {
			System.out.print("e");
		}
	}
}
```

</CodeGroup>

Notice how Go prioritizes brevity, Rust requires explicit acknowledgment of infinite loops with `-> !`, and Java demands ceremony even for the simplest programs.

## The Universal Truth

Every programming language, no matter how different, can express the same computation. This is the Church-Turing thesis made manifest: **all Turing-complete languages are fundamentally equivalent**.

<Tip>
  'e' proves that beneath syntax and paradigms, all programming languages speak the same universal language of computation.
</Tip>

## The Infinite Loop Paradox

Why does 'e' print infinitely? Because:

1. **Infinite loops are fundamental** - They represent continuous computation, servers, event loops, and the heart of reactive systems
2. **Termination is arbitrary** - Choosing when to stop would introduce arbitrary complexity
3. **Infinity is honest** - An infinite program acknowledges that computation continues until interrupted

## Final Thoughts

> "Programs must be written for people to read, and only incidentally for machines to execute." — Harold Abelson

The 'e' project takes this to its logical conclusion: if code is for humans, then the clearest code is the simplest code. And the simplest code that demonstrates a language's essence is one that prints 'e' forever.

<Note>
  In a world of complexity, 'e' is a return to first principles. It asks: **what is the smallest program that matters?**
  
  The answer is elegantly simple: 'e'.
</Note>
