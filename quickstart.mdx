# Quickstart Guide

Get up and running with 'e' in just a few steps. This guide will walk you through running your first 'e' program across multiple languages.

<Note>
  All source files are located in the `source/` directory. Each file follows the naming convention `e.<extension>`.
</Note>

## Prerequisites

To run 'e' programs, you'll need the appropriate language runtime or compiler installed. Here are some common ones:

- **Python 3**: `python3`
- **Node.js**: `node`
- **Go**: `go`
- **Rust**: `rustc` and `cargo`
- **C/C++**: `gcc` or `clang`
- **Ruby**: `ruby`
- **Java**: `javac` and `java`

## Running Your First 'e'

<Steps>
  <Step title="Choose a Language">
    Pick your favorite programming language. We'll start with Python as an example:
    
    ```python e.py
    import sys

    try:
        while 'e':
            sys.stdout.write('e')
    except KeyboardInterrupt as e:
        sys.exit('e')
    ```
  </Step>

  <Step title="Run the Program">
    Execute the program using your language's runtime:
    
    ```bash
    python3 source/e.py
    ```
    
    You should see an infinite stream of 'e' characters:
    ```
    eeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeeee...
    ```
  </Step>

  <Step title="Stop the Program">
    Press `Ctrl+C` to stop the infinite 'e' stream.
    
    <Tip>
      Most 'e' programs are designed to run forever. Always use `Ctrl+C` to terminate them gracefully.
    </Tip>
  </Step>
</Steps>

## Try More Languages

Once you've successfully run your first 'e', try exploring other implementations:

### JavaScript

```bash
node source/e.js
```

```javascript e.js
while ('e') console.log('e');
```

### Go

```bash
go run source/e.go
```

```go e.go
package main

func main() {
	for {
		print("e")
	}
}
```

### Rust

```bash
rustc source/e.rs -o e_binary && ./e_binary
```

```rust e.rs
#![feature(main)]
#[cfg(not(feature = "functional"))]
#[main]
fn e() -> ! {
    loop {
        print!("e");
    }
}
```

### Ruby

```bash
ruby source/e.rb
```

```ruby e.rb
loop do
  print 'e'
end
```

### Shell Script

```bash
bash source/e.sh
```

```bash e.sh
#!/bin/sh
yes e | tr -d '\n'
```

## Docker

Run 'e' in a container:

```dockerfile e.Dockerfile
FROM busybox

RUN seq 1 10000 | while read -r e; do echo -ne 'e'; done; echo e;

CMD strings /dev/urandom | grep -oE 'e' | tr -d '\n'
```

```bash
docker build -f source/e.Dockerfile -t e:latest .
docker run e:latest
```

```yaml e.yml

CREATE DATABASE e;
USE e;

DELIMITER |
CREATE PROCEDURE e()
BEGIN
    WHILE 'e' = 'e' DO
        SELECT REPEAT("e",RAND()*100) AS e;
    END WHILE;
END|
DELIMITER ;

CALL e();
```

</CodeGroup>

## Common Issues

<AccordionGroup>
  <Accordion title="The program won't stop!">
    This is by design. All 'e' programs run infinitely. Use `Ctrl+C` to stop execution.
  </Accordion>

  <Accordion title="I get a 'command not found' error">
    Make sure you have the appropriate language runtime or compiler installed for the 'e' implementation you're trying to run.
  </Accordion>

  <Accordion title="Can I redirect output to a file?">
    Yes! Try `python3 source/e.py > output.txt` but be warned: the file will grow indefinitely until you stop the program.
  </Accordion>
</AccordionGroup>

<Tip>
  **Pro tip**: Use output redirection and `head` to capture a finite amount of 'e's:
  
  ```bash
  python3 source/e.py | head -c 1000
  ```
  
  This will print exactly 1000 'e' characters and then stop.
</Tip>
